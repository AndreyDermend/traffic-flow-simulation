Engineering process

The story behind how this project was built, from observation to deployment.

1. Real-world data collection

Observed actual streets, estimated queue capacity, and counted vehicle flow over short windows to ground the simulation in something real.

2. C systems design

Modeled streets, cars, queues, lights, and routing rules in C using structs, linked-list queues, timing logic, and probabilistic movement.

3. Team implementation

Split the work across simulation functions such as initialization, queue movement, routing, timing, printing, and debugging, then integrated the pieces together.

4. Edge-case testing

Tested low and high represented seconds, full queues, empty queues, red-light blocking, stop-sign streets, and destination-overflow behavior.

5. Full-stack rebuild

Wrapped the compiled C engine in a Python API and rebuilt the output as a browser visualizer so the simulation is easier to run, explain, and show in a portfolio.
