# Screenshot checklist

Add screenshots here before pushing the repo publicly.

Recommended captures:

1. **Homepage / dashboard**
   - Full browser view of the frontend after a simulation run
   - Include summary cards and at least a few street cards

2. **Playback slider**
   - Show the tick slider in use so the project looks interactive

3. **Street detail state**
   - Capture at least one moment where one street is clearly congested

4. **Code architecture shot**
   - Optional: a screenshot of the repo tree or diagram for your portfolio case study

5. **Real-world context image**
   - Optional: a photo, map crop, or sketch of the observed intersection area
   - Only include if you are comfortable sharing it publicly
